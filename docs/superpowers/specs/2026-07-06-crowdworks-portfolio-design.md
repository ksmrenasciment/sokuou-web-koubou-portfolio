# クラウドワークス営業用ポートフォリオサイト 設計書

## 目的

ホームページ制作の受託事業（クラウドワークス経由）を開始するにあたり、実績ゼロの状態で信頼を獲得するためのポートフォリオサイトを制作する。クラウドワークスの提案文から個別事例へ直接リンクできる「営業ツール」として機能させることを最優先する。

## 全体方針

- 実績はまだ無いため、5件のダミー案件（架空案件）を制作し、実力の証明とする
- サイトの見せ方は「制作実績中心」。プロフィールは簡潔にし、実績の厚みで信頼を作る
- 自己紹介文のみ実話ベース（経歴：IT企業 → リクルート → 現職・大手福祉企業）、他のコンテンツ（屋号・ダミー案件）は仮のものとして作成し、後から差し替え可能な構成にする

## アーキテクチャ / 技術構成

- **フレームワーク**: Next.js（静的書き出し / App Router）+ Tailwind CSS
- **ホスティング**: Vercel の無料サブドメインでまず公開。将来的に独自ドメインへ切替可能な構成にする（Next.jsのカスタムドメイン設定のみで完結させ、コード側の変更を不要にする）
- **WordPress案件（ダミー#2）**: 実際にWordPress環境は構築しない。静的HTML/CSSでWordPressテーマ風の見た目を再現し、ケーススタディページ内に組み込む。ケーススタディの文章では「WordPressでの納品を想定した実装」であることを明記し、実際にWordPressが稼働しているという誤解を与えない
- **画像**: 実在人物の顔写真や実在企業のロゴは使用しない。フリー素材・抽象的なビジュアル・配色主体のデザインで構成し、架空案件であることが伝わる範囲で仕上げる

## サイト構成（ハイブリッド型）

```
/ (トップページ)
  - ヒーロー（屋号・キャッチコピー）
  - 自己紹介（実話ベース）
  - 強み3点
  - 実績サマリー（5件のカード、各ケーススタディへリンク）
  - CTA（クラウドワークスでの問い合わせ導線）

/works/[slug] （ケーススタディ詳細 × 5）
  - 課題 → 対応内容 → 成果 の型で統一
  - 使用技術・制作期間・体制（1人）を明記
```

## コンテンツ計画

### 屋号（仮）

「即応Web工房」。強みの一つである対応速度を反映した仮称。事業が固まった段階で正式名称に差し替え可能なよう、ロゴはテキストロゴ（画像埋め込みではない）で実装する。

### 自己紹介文（実話ベース・案）

> 社会人1年目にIT企業でキャリアをスタートし、その後リクルートで営業・企画を経験。現在は大手福祉企業に勤務しながら、AIを活用したホームページ制作を副業として行っています。IT・営業・福祉と異なる業種を経験してきたからこそ、「技術だけでなく、事業側の視点に立った提案」を強みにしています。

- 現職（大手福祉企業）に触れるのは自己紹介文の中で1回程度に留め、「福祉専門」という印象を持たれないようにする（強み3点目「幅広い技術対応」と併記してバランスを取る）

### 強み（3点）

1. 対応の速さ・コミュニケーション
2. AI活用による高速・低コスト制作
3. 幅広い技術対応（静的サイト〜WordPress）

### ダミー案件（5件）

| # | 案件 | タイプ | ポイント |

| 1 | 地域密着デイサービスのコーポレートサイト | 静的サイト | 福祉業界の内部事情を理解した「家族・利用者にわかりやすい導線設計」を強みとして織り込む（福祉色は強く出しすぎない） |

業種を福祉・飲食・LP・教育・建設に分散させ、特定業界への特化ではなく「幅広い技術対応」を裏付ける構成にする。

## 視覚デザインの方向性

**D. ミニマル・洗練**をベースに、差し色を1色追加する。

- ベース: ホワイト `#FFFFFF` ／ ライトグレー `#F4F4F4` ／ チャコールブラック `#1A1A1A`（見出し・本文）
- 差し色: ディープインディゴ `#2F3E63`（CTAボタン・リンク・アクセントラインのみに使用）

モノトーンで洗練された印象を保ちつつ、CTAなど強調箇所にのみインディゴを使うことで信頼感も補強する。

## テスト・検証方針

- 主要ブラウザ（Chrome/Safari）でのレイアウト崩れ確認
- モバイル幅（375px程度）でのレスポンシブ表示確認
- 各ケーススタディページへのリンク導線が正しく機能すること
- Vercelへのデプロイ後、公開URLで実際にアクセスできることを確認

## スコープ外（今回やらないこと）

- 独自ドメインの取得・設定
- 実際のWordPress環境構築（ダミー#2は静的再現のみ）
- お問い合わせフォームのバックエンド実装（クラウドワークスへの導線リンクのみで足りるため）
- 多言語対応
